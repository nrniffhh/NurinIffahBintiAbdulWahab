Learning Labs / Hands-On Lab Sessions

Lab 1

UI Layouts with Jetpack Compose

Explored fundamental Compose layout components — Column, Row, and Box — to build structured and responsive user interfaces.

Reflection: In Lab 1, I learned the fundamentals of Jetpack Compose layouts using components such as Column, Row, and Box. This lab helped me understand how to arrange UI elements effectively and create a structured user interface. It provided the vital spatial engineering foundation for developing the screens inside my DailyFit application.
Lab 2

Interaction & State Management

Built interactive UIs using TextField and Button components, managing dynamic state mutations natively through standard modern Android tools.

Reflection: In Lab 2, I learned how to create interactive user interfaces using TextField, Button, and state management with remember and mutableStateOf mechanics. I gained a better understanding of how user input can update the UI dynamically through recomposition. This knowledge was useful when implementing immediate fitness user interactions in my application.
Lab 3

Material Design & Theming

Applied Material Design principles — custom themes, card components, and smooth animations — to produce a polished, consistent visual experience.

Reflection: In Lab 3, I learned how to apply Material Design principles by creating a custom theme, using cards, and adding animations. I understood the importance of consistent colors, typography, and user experience in application design. This lab helped improve the visual appearance and usability of my app.
Lab 4

Navigation & ViewModel

Implemented multi-screen view management with standard NavHost routers, and managed shared data pipelines via ViewModel state mechanisms.

Reflection: In Lab 4, I learned how to implement navigation between multiple screens using NavHost and routes. I also explored ViewModel, UiState, and StateFlow for managing shared data across screens. Although these concepts were challenging at first, they helped me understand how modern Android applications manage navigation and state.
Lab 5

Room Database & Persistence

Configured Room Database engines for persistent local state tracking, seamlessly integrating reactive DAO contracts, repository interfaces, and state-holding viewmodels.

Reflection: In Lab 5, I learned how to use Room Database for local data persistence. I gained experience with DAO, Database, Repository, and ViewModel integration to store and retrieve data efficiently. This lab helped me understand how applications can maintain historical workout data even after being closed and reopened.
